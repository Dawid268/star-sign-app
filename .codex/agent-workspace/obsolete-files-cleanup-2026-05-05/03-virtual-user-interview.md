# Virtual user interview

## Perspektywa użytkownika projektu

Jako osoba pracująca z repo chcę widzieć tylko aktualne raporty i dokumenty, żeby nie podejmować decyzji na podstawie starych wniosków.

## Obawy

- Usunięcie aktualnego runbooka albo checklisty mogłoby utrudnić release.
- Usunięcie starych artefaktów agentów jest akceptowalne, jeśli finalne reusable knowledge zostało zapisane w Serenie albo bieżącym raporcie.

## Polska konkluzja

Najbardziej przeszkadzają stare workspace'y agentów i jawnie historyczne audyty. Dokumentację operacyjną lepiej zostawić.
