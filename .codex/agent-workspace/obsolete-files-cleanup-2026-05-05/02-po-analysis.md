# PO analysis

## Cel biznesowy

Repo ma być czytelniejsze i mniej obciążone starymi raportami roboczymi. Deweloperzy i właściciel produktu nie powinni mylić bieżących decyzji z historycznymi audytami.

## Must-have

- Usunąć stare artefakty agentów.
- Usunąć plik, który sam oznacza się jako nieaktualny.
- Nie kasować dokumentacji operacyjnej, która może być nadal używana przy wdrożeniu.

## Polska konkluzja

Wartość tej fazy to redukcja szumu, nie refaktor funkcji. Kasowanie musi być selektywne.
